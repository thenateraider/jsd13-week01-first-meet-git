### Git Version Control Guide
Follow these standard commands to save, commit, and push your changes to a remote repository:

1. **Check Workspace Status**
   See which files have been modified, created, or deleted.
   ```bash
   git status
   ```

2. **Stage Your Changes**
   Prepare files for the next snapshot.
   * To stage a single file (e.g., `README.md`):
     ```bash
     git add README.md
     ```
   * To stage all modified and new files:
     ```bash
     git add .
     ```

3. **Commit Changes**
   Create a local snapshot of staged files with a descriptive summary.
   ```bash
   git commit -m "Create multilingual README and Git guide"
   ```

4. **Push to Remote Repository**
   Upload local commits to the remote hosting platform (e.g., GitHub).
   ```bash
   git push origin main
   ```
   *(Note: Replace `main` with your current branch name if different, such as `master` or `dev`).*

### คู่มือการใช้คำสั่ง Git
โปรดใช้คำสั่ง Git เหล่านี้ตามลำดับเพื่อบันทึกประวัติการพัฒนาและอัปเดตงานไปยังเซิร์ฟเวอร์ปลายทาง (เช่น GitHub):

1. **ตรวจสอบสถานะปัจจุบัน (Git Status)**
   เช็กว่าไฟล์ใดบ้างที่มีการแก้ไข สร้างขึ้นใหม่ หรือถูกลบออกไป
   ```bash
   git status
   ```

2. **เตรียมบันทึกข้อมูล (Git Add)**
   เตรียมไฟล์เข้าสู่ระบบเพื่อพร้อม Commit
   * กรณีต้องการเลือกเฉพาะไฟล์ (เช่น `README.md`):
     ```bash
     git add README.md
     ```
   * กรณีต้องการเพิ่มการแก้ไขทั้งหมดในโฟลเดอร์:
     ```bash
     git add .
     ```

3. **บันทึกประวัติความเปลี่ยนแปลง (Git Commit)**
   บันทึกภาพรวมความเปลี่ยนแปลงที่เกิดขึ้นในเครื่องพร้อมระบุข้อความอธิบาย
   ```bash
   git commit -m "เพิ่มไฟล์ README สามภาษา และคู่มือการใช้งาน Git"
   ```

4. **อัปโหลดข้อมูลสู่เซิร์ฟเวอร์หลัก (Git Push)**
   ส่งประวัติที่บันทึกแล้วไปยังพื้นที่เก็บข้อมูลออนไลน์ปลายทาง
   ```bash
   git push origin main
   ```
   *(หมายเหตุ: ให้เปลี่ยนคำว่า `main` เป็นชื่อบรันช์ปัจจุบันของคุณ เช่น `master` หรือ `dev` หากมีความแตกต่าง)*
